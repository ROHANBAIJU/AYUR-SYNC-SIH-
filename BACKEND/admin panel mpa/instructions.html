Problem Statement: 25026

Title: Develop API code to integrate NAMASTE and/or the ICD‑11 via TM2 into EMRs compliant with India’s EHR Standards.

Read full description

Background — India’s Ayush sector is transitioning to interoperable digital systems. This project bridges NAMASTE codes, WHO International Terminologies for Ayurveda, and ICD‑11 TM2/biomedicine in FHIR R4 EMRs with OAuth, consent and audit readiness.

Objective — Provide a lightweight terminology micro‑service or FHIR plugin: CodeSystem for NAMASTE/International Terminologies, ConceptMap for ICD‑11 TM2 & biomedicine, lookup and translate operations, and secure Bundle ingest.

  • Ingest NAMASTE CSV; generate FHIR CodeSystem + ConceptMap
  • Fetch TM2/biomed updates from WHO ICD‑API and merge
  • Simple UI/CLI to search, map, and assemble a FHIR ProblemList
  • Versioning and consent metadata aligned with India’s 2016 EHR Standards

Expected Solution — FHIR R4 compliant micro‑service offering lookup, translate, double coding, and secure ingestion; suitable for EMR integration and analytics.

Org: Ministry of Ayush · Dept: AIIA · Theme: MedTech / BioTech / HealthTech

Admin Interfaces Overview

This panel enables human‑in‑the‑loop verification of AI‑generated mappings across Ayurveda, Siddha and Unani.

Analytics

Visualizes real usage from the API access log: requests over time, top paths, live test calls, and India map of diagnosis events. Use it to validate clinician‑facing endpoints and observe how verified mappings impact results.

New Suggestions (Curation)

  • Each card groups suggestions by an ICD‑11 disease. You’ll see suggestions per system with AI justification and confidence.
  • Select one Primary per system; add optional Aliases. You can reject incorrect suggestions with reasons:
  • — Incorrect mapping: term is valid but mapped to wrong ICD → goes to Correction Queue.
  • — Orphan: term is invalid/misspelled → goes to Orphanage.
  • Click Save Curation Changes. Approved items move to Master Map (Staged).
  • Tip: After reset, the panel auto‑polls until fresh suggestions arrive.

Master Map

Shows staged mappings ready to be promoted to verified. You can edit the Primary/Aliases per system and then Commit to Verified DB. Verified mappings are used by clinician endpoints (/lookup and /translate).

ICD‑11 Disease List

Maintains the master list of ICD‑11 diseases for curation. Add new entries where needed (e.g., when remapping rejected items to a new disease).

Rejected Mappings

  • Correction Queue: terms to be remapped to a different ICD‑11 disease.
  • Orphanage: invalid or unrecognized terms for later cleanup and analysis.
  • You can Remap a rejected term: pick destination ICD or add a new one; AI re‑analysis updates justification/confidence.
